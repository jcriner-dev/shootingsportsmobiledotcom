[2022.3.2] - 2022-09-01

Added

  • Added chrono feature to server and scoring app.
    • The scoring app works. See the help file.
    • The server side will use chrono and equipment check data only if present when uploaded.
    • Failed chrono gets a DQ and failed equipment check gets a DNF per IDPA rules.
    • Chrono and equipment check are include in:
      • Match reports.
      • Match performance promotions.
      • Season standings.

[2022.3.1] - 2022-08-19

Added

  • Added beta chrono feature to server and scoring app.
    • The scoring app works. There is no help yet.
    • The server side simply prints out failed chrono or equipment checks when you upload.
    • Therefore, you must manually apply DNF or DQ due to chrono or equipment check.
    • The plan for integrating chrono on the server side is TBD.

[2022.2.3] - 2022-08-11

Added

  • Added custom sql files for the 2022 MI State Match.

Changed

  • Change all "Custom Properties" defaults to "Yes".
  • Changed update_match_dnf.
    • Match DNF is calculated based on either the specified stage count in the match description, or the most common stage count for all competitors when not specified.
    • Previously, it was recalculated when running match results reports or exports.
    • Now, it is recalculated on every upload. This allows for chrono or equipment DNF's to be manually applied after all uploading is done.
  • Changed font size calculation for pdf reports to better use available space with greater than 10 stages.

Fixed

  • Fixed most_accurate to require hnt = 0.
  • Fixed display of dnf and dq in match results and match exports.
  • Fixed bug in patch_db when comparing versions.

Other

  • Scoring versions >= 2022.2.2 are fully compatible with this admin and server version.

[2019.1.1] - 2019-04-05

Changed

  • Changed from "semver.org" to "calver.org" versioning.
    • Rationale
      • Project should have been past version 1.0 over five years ago.
      • Project did not follow strict semver regarding breaking changes.
      • Rules tend to be season oriented.
    • Format
      • YYYY.version.patch
      • YYYY = four (4) digit year.
      • version = incremental integer version for the year.
        • Used for feature changes or improvements.
        • MAY or MAY NOT be backward compatible with the same YYYY.
      • patch = incremental integer bugfix.
        • Used for bugfixes only.
        • MUST be backwards compatible with the same YYYY.version.

Fixed

  • Fixed filename bug in idpahq.py when using --diff-status on the command line.

[0.5.1] - 2017-02-11

Added

  • Added "down_value" to server config (defaults to 1.0 2017 rules). Score is calculated as PD * down_value.
  • Added option to get results as a webpage or PDF download.
  • Added "best of" feature to season standings to allow throwouts.
  • Added "XXX" = "Anything Goes" division as a catchall.
  • Added import and editor for arbitrary unofficial divisions (think SPD, PCC, BBQ, ETC).
  • Added BUG-R, BUG-S to make_model.csv to allow for subdivisions via makes.
    • Use the make/model/caliber feature at signup to identify subdivisions.
    • Results are still grouped by division, but you can use this method to identify subdivisions.
  • Added First Responder (FP) category. We have been using it for years locally and figured others may want to as well.

Changed

  • Changed scoring formula for Points Down from PD * 0.5 to PD * 1.0 (2017 Rules).
  • Changed scoring formula and scoring program to add Flagrant Penalty FP * 10.0 (2017 Rules).
  • Changed scoring formula and scoring program to remove Failure To Neutralize FTN * 5.0 (2017 Rules).
  • Changed number of strings and rounds for classifier from 7, 4, 3 (90) to 3, 3, 2 (72) (2017 Rules).
  • Changed classification times for new classifier (2017 Rules).

Removed

  • Remove timezone specification.
  • Removed Not For Competition (NFC) Division.
  • Removed FTN button from scoring app (2017 Rules).

Other

  • There are a number of breaking changes in this version.
  • You can still generate reports from existing matches and classifiers, but any attempt to change (admin/stage edit) and / or re-upload (scoring/upload) existing scores will lead to problems. If you need to do that, you should be using the versions that generated the scores in the first place.
  • Category differences from 2017 Rulebook:
    • M6 Competitors may choose ONE of the following to add to their registration...
      • This change is probably to prevent a single shooter from being acknowledged in multiple categories.
      • The software currently associates "categories" with shooters, not with entries, because categories do not tend to change match to match.
      • If you believe that someone who served their fellow citizens (Military, Law Enforcement, First Responder) and their fellow shooters (Safety Officer) deserves to be recognized as both, continue to check as many categories as are appropriate for each shooter.
      • WORKAROUND: If you want to strictly comply with the 2017 rules, simply make sure that each shooter has only one category checked at any given time. The score reports will reflect the category checked at the time the report is generated.
    • M6.8 Military & M6.9 Law Enforcement are now "active" only. This change is probably to force consistency with the one category change above.
      • We strongly believe that anyone who took an oath to serve their fellow citizens deserves to be recognized no matter how long they were able to serve. As such, our default category descriptions for Military, Law Enforcement, and First Responder will always reflect that. If a Veteran can outshoot an Active Member, then we would like to recognize them.
      • WORKAROUND: If you want to strictly comply with the 2017 rules and have your category descriptions consistent with the rulebook there are two options:
        1. Edit your categories.csv file before you start the program for the first time and create your database.
        2. Contact us and we will give you simple and straightforward instructions on how to edit the category descriptions in your existing database. It isn't hard and we are happy to help.
